# 第 1 章：脚本 mod 的范围

第 0 章已经把层级摆出来了。

现在只看一件事：BML 脚本 mod 能做什么，先不做什么。

前几章只改脚本文件。它在游戏运行时由 BMLPlus 加载，不保存关卡内容，也不替换原版 NMO/CMO 文件。

本章结束时，要能回答：

```text
我现在写的脚本会影响哪里？
它暂时不应该碰哪里？
```

## 脚本 mod 是运行时代码

第 0 章已经给过加载流程。这里先记住结果：BMLPlus 会在游戏运行时调用脚本里的函数。

所以脚本 mod 的核心工作是在游戏运行时执行代码。

## 关卡文件负责关卡内容

关卡内容来自原版文件，例如：

- 地板、轨道、机关；
- 道具、检查点、复活点；
- 起点、终点、小节；
- 关卡里的对象、组和一些数据表。

这些内容通常在关卡 NMO 里。

脚本 mod 前期不去改这些文件。

## 前几章会做什么

前几章只做 BML 层的基础能力：

| 能力 | 用来做什么 |
| --- | --- |
| 日志 | 确认脚本加载、记录状态 |
| ImGui 调试窗口 | 在游戏里显示脚本状态 |
| 命令 | 手动触发脚本函数 |
| 配置 | 保存脚本自己的设置 |
| Timer | 延迟执行或低频执行 |

这些能力不会改变关卡文件。

它们的作用是先搭一个能跑、能看、能操作的脚本外壳。

后面读取对象、组、DataArray 时，也会用到这个外壳来显示结果和触发扫描。

## 现在不做什么

下面这些内容先不碰：

- 修改 `Gameplay.nmo`；
- 修改 `Levelinit.nmo`；
- 写 `CurrentLevel`；
- 移动当前正在玩的球；
- 修改检查点、复活点流程；
- 修改原版行为图；
- 改原版物理参数。

这些内容牵涉原版流程。后面先讲清楚对象、组、DataArray、行为图和物理事件，再回到修改问题。

现在先把脚本本身跑稳。

## 开始前检查

在 Ballance 目录里确认这些文件存在：

| 文件 | 说明 |
| --- | --- |
| `BuildingBlocks/BMLPlus.dll` | BMLPlus |
| `BuildingBlocks/AngelScript.dll` | CKAS / AngelScript 支持 |
| `3D Entities/Gameplay.nmo` | 原版玩法逻辑 |
| `3D Entities/Levelinit.nmo` | 原版关卡初始化 |
| `3D Entities/Level/Level_01.NMO` | 第一关内容 |

再确认后面要放脚本的目录存在：

```text
ModLoader/Mods
```

如果没有，后面创建脚本时再创建。
